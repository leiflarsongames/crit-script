# Develop an intended mental model of CritScript Graph execution
Break down how you want to have the user understand the flow of execution. This will allow them to accurately diagnose errors in their CritScript graphs!
To keep the barrier for entry as low as possible for visual scripting:
* Don't expose what isn't necessary to the user!
* Radically simplify while being as helpful as possible!

# CritScript errors stemming from Python implementations
For errors in user-made python functions, it's unavoidable that Python gets involved. 
Be as helpful as physically possible by:
* Giving examples of what might be wrong, and instructions to correct it. Include code examples!
* Make the error as transparent as possible
* Keep formatting on those errors as consistent as possible so skills like "reading errors" carries over as much as possible between different errors!
